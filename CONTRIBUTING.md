# Contributing to InVivoSegment

Thank you for your interest in contributing to InVivoSegment! We welcome contributions from the community.

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue on GitHub with:

- A clear description of the problem
- Steps to reproduce the issue
- Your environment (OS, Python version, etc.)
- Any error messages or logs

### Suggesting Enhancements

We're open to suggestions! Please create an issue with:

- A clear description of the enhancement
- Use cases and examples
- Any relevant references or implementation ideas

### Code Contributions

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Development Setup

```bash
# Clone the repository
git clone https://github.com/bearererlab/InVivoSegment.git
cd InVivoSegment

# Create a virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install the package in development mode
pip install -e .
```

## Code Style

- Follow PEP 8 conventions
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Include comments for complex logic

## Pull Request Process

1. Update the README.md with any new features or changes
2. Ensure your code follows the project's style guidelines
3. Write clear commit messages
4. Reference any related issues in your PR description

## Questions?

Feel free to open an issue to ask questions or start a discussion about the project.

Thank you for contributing!
